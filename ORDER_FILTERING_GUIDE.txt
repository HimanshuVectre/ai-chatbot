========== ORDER FILTERING - AGENT GUIDANCE ==========

ISSUE IDENTIFIED & FIXED
─────────────────────────

Problem: Agent was showing all orders instead of filtering for specific invoices or last N orders.

Root Cause: Agent instructions were unclear about when/how to use tool parameters.

Solution: Enhanced instructions + tool description to explicitly map queries to parameters.

---

QUERY → TOOL PARAMETER MAPPING
──────────────────────────────

CUSTOMER QUERY              | TOOL PARAMETER              | RESULT
────────────────────────────────────────────────────────────────────
"Show me INV-1006"          | invoice_id="INV-1006"       | 1 specific order
"Order details INV-0009"    | invoice_id="INV-0009"       | 1 specific order
"What's my last order?"     | limit=1                     | Last 1 order
"My last 2 orders"          | limit=2                     | Last 2 orders
"Last 5 orders"             | limit=5                     | Last 5 orders
"Show my orders"            | (default limit=20)          | First 20 orders
"All my orders"             | (no parameters)             | All orders (up to 20)
"I don't know the order ID" | (default limit=20)          | Show recent orders

AGENT INSTRUCTIONS - UPDATED
─────────────────────────────

Key additions to SalesAssistant.php:

1. "WHEN USING ListOrderToolForUser, apply EXACT filtering:"
2. "If customer asks for specific invoice ID (e.g., "INV-0009"), use invoice_id parameter ONLY."
3. "If customer asks for "my last N orders", use limit=N parameter."
4. "CRITICAL: Do NOT fetch all orders and then filter client-side. Use tool parameters."

Examples provided:
• "Show me order INV-0009" → Use invoice_id="INV-0009"
• "What are my last 2 orders?" → Use limit=2
• "Show my orders" → Use default

---

TOOL DESCRIPTION - ENHANCED
───────────────────────────

Updated ListOrderToolForUser description to explicitly state:

"Parameters:
(1) invoice_id (string, optional) - Search for specific order by invoice ID
(2) limit (integer, optional) - Max number of recent orders (1-50, default 20)

Examples:
- invoice_id="INV-0009" returns 1 specific order
- limit=2 returns last 2 orders
- no parameters returns first 20 orders"
